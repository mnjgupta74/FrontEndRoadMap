इंटरनेट प्रोटोकॉल्स (Internet Protocols)

प्रोटोकॉल क्या है? इंटरनेट प्रोटोकॉल्स डेटा आदान-प्रदान के लिए नियमों और मानकों का एक सेट है। ये नियम बताते हैं कि नेटवर्क पर डिवाइसेस आपस में कैसे संवाद करें।

1. TCP/IP (Transmission Control Protocol/Internet Protocol)

TCP/IP इंटरनेट का मूल प्रोटोकॉल है। यह आधुनिक इंटरनेट की रीढ़ है और निम्नलिखित लेयर्स से बना है:

  • एप्लिकेशन लेयर: HTTP, FTP, SMTP, DNS आदि
  • ट्रांसपोर्ट लेयर: TCP, UDP
  • इंटरनेट लेयर: IP, ICMP
  • नेटवर्क इंटरफेस लेयर: Ethernet, Wi-Fi आदि

उदाहरण - TCP/IP कैसे काम करता है:

जब आप वेबसाइट खोलते हैं:

  1. एप्लिकेशन लेयर: आपका ब्राउज़र HTTP रिक्वेस्ट तैयार करता है
  2. ट्रांसपोर्ट लेयर: TCP डेटा को सेगमेंट्स में विभाजित करता है और सीक्वेंस नंबर जोड़ता है
  3. इंटरनेट लेयर: IP प्रत्येक सेगमेंट को पैकेट्स में बदलता है और स्रोत/गंतव्य IP एड्रेस जोड़ता है
  4. नेटवर्क इंटरफेस लेयर: पैकेट्स को फिजिकल नेटवर्क पर भेजता है
  5. सर्वर पर, डेटा इन्हीं लेयर्स से विपरीत क्रम में गुजरता है

2. HTTP और HTTPS (HyperText Transfer Protocol)

HTTP पोर्ट 80

HTTP वेब पेजेस और रिसोर्सेज (इमेजेस, CSS, JavaScript) को ट्रांसफर करने के लिए उपयोग किया जाने वाला प्रोटोकॉल है।

उदाहरण - HTTP रिक्वेस्ट और रिस्पांस:

HTTP रिक्वेस्ट:

GET /index.html HTTP/1.1
Host: www.example.com
User-Agent: Mozilla/5.0
Accept: text/html

HTTP रिस्पांस:

HTTP/1.1 200 OK
Date: Mon, 23 May 2023 22:38:34 GMT
Content-Type: text/html
Content-Length: 138

<html>
<body>
<h1>Hello, World!</h1>
</body>
</html>

HTTPS पोर्ट 443

HTTPS, HTTP का सुरक्षित संस्करण है। यह SSL/TLS का उपयोग करके डेटा को एन्क्रिप्ट करता है।

उदाहरण - HTTPS कैसे काम करता है:

  1. जब आप https://bank.com खोलते हैं, आपका ब्राउज़र और बैंक का सर्वर SSL/TLS हैंडशेक करते हैं
  2. सर्वर अपना SSL सर्टिफिकेट भेजता है
  3. ब्राउज़र सर्टिफिकेट को वेरिफाई करता है और एक सिमेट्रिक की (session key) जनरेट करता है
  4. ब्राउज़र सर्वर के पब्लिक की से session key को एन्क्रिप्ट करके सर्वर को भेजता है
  5. सर्वर अपनी प्राइवेट की से इसे डिक्रिप्ट करता है
  6. अब दोनों के पास एक शेयर्ड सिक्रेट है, जिससे एन्क्रिप्टेड कम्युनिकेशन होता है

इस तरह, अगर कोई आपके डेटा को इंटरसेप्ट करता है, तो वह एन्क्रिप्शन के कारण उसे पढ़ नहीं सकता।

3. FTP (File Transfer Protocol)

FTP पोर्ट 21 (कंट्रोल), 20 (डेटा)

FTP सर्वर और क्लाइंट के बीच फाइल्स ट्रांसफर करने के लिए उपयोग किया जाता है।

उदाहरण - FTP द्वारा फाइल अपलोड/डाउनलोड:

FTP कमांड्स:

USER username
PASS password
PWD (वर्तमान डायरेक्टरी दिखाना)
LIST (फाइल्स की लिस्ट देखना)
RETR filename (फाइल डाउनलोड करना)
STOR filename (फाइल अपलोड करना)

प्रैक्टिकल उदाहरण:

वेब डेवलपर अक्सर FTP का उपयोग करके वेबसाइट फाइल्स (HTML, CSS, JavaScript) को वेब होस्टिंग सर्वर पर अपलोड करते हैं। FileZilla जैसे FTP क्लाइंट्स इस प्रक्रिया को आसान बनाते हैं।

SFTP (Secure File Transfer Protocol) पोर्ट 22

SFTP, FTP का सुरक्षित संस्करण है जो SSH (Secure Shell) का उपयोग करता है।

SFTP में सभी फाइल ट्रांसफर एन्क्रिप्टेड होते हैं, जबकि FTP में डेटा प्लेन टेक्स्ट में भेजा जाता है।

4. SMTP, POP3, और IMAP (ईमेल प्रोटोकॉल्स)

SMTP (Simple Mail Transfer Protocol) पोर्ट 25

SMTP ईमेल भेजने के लिए उपयोग किया जाता है।

उदाहरण - SMTP द्वारा ईमेल भेजना:

MAIL FROM: <sender@example.com>
RCPT TO: <recipient@example.com>
DATA
Subject: Hello

This is a test email.
.
QUIT

जब आप जीमेल पर Send बटन दबाते हैं, तो आपका ईमेल क्लाइंट SMTP सर्वर से कनेक्ट करता है और उपरोक्त जैसे कमांड्स का उपयोग करके मेसेज भेजता है।

POP3 (Post Office Protocol) पोर्ट 110

POP3 ईमेल डाउनलोड करने के लिए उपयोग किया जाता है। यह मेसेज को सर्वर से डाउनलोड करके डिलीट कर देता है।

उदाहरण - POP3 द्वारा ईमेल प्राप्त करना:

USER username
PASS password
STAT (मेसेज की संख्या और साइज)
LIST (मेसेज की सूची)
RETR 1 (पहला मेसेज डाउनलोड करना)
DELE 1 (पहला मेसेज डिलीट करना)
QUIT

IMAP (Internet Message Access Protocol) पोर्ट 143

IMAP भी ईमेल प्राप्त करने के लिए है, लेकिन यह मेसेज को सर्वर पर रखता है और मल्टीपल डिवाइसेस से एक्सेस की अनुमति देता है।

उदाहरण - IMAP vs POP3:

POP3 सिनेरियो: आप ऑफिस के कंप्यूटर से ईमेल चेक करते हैं और POP3 द्वारा सभी मेसेज डाउनलोड करते हैं। घर पहुंचकर लैपटॉप से चेक करते हैं, लेकिन वहां कोई मेसेज नहीं दिखता क्योंकि वे सर्वर से डिलीट हो चुके हैं।

IMAP सिनेरियो: आप ऑफिस में ईमेल चेक करते हैं, कुछ मेसेज पढ़ते हैं और फोल्डर में ऑर्गेनाइज करते हैं। घर पहुंचकर लैपटॉप से चेक करते हैं, और सभी मेसेज और फोल्डर्स वैसे ही दिखते हैं, क्योंकि IMAP सभी चेंजेस को सर्वर पर सिंक रखता है।

5. DNS (Domain Name System)

DNS पोर्ट 53

DNS डोमेन नेम्स (जैसे google.com) को IP एड्रेस में बदलता है।

उदाहरण - DNS लुकअप प्रक्रिया:

  1. आप ब्राउज़र में www.example.com टाइप करते हैं
  2. आपका कंप्यूटर लोकल DNS रिज़ॉल्वर को क्वेरी भेजता है
  3. DNS रिज़ॉल्वर रूट DNS सर्वर से पूछता है
  4. रूट सर्वर कहता है: ".com DNS सर्वर्स से पूछो"
  5. .com DNS सर्वर कहता है: "example.com के नेमसर्वर्स से पूछो"
  6. example.com का DNS सर्वर फाइनल आंसर देता है: "www.example.com का IP 93.184.216.34 है"
  7. अब आपका ब्राउज़र 93.184.216.34 से कनेक्ट करता है

प्रैक्टिकल उदाहरण: आप कमांड प्रॉम्प्ट या टर्मिनल में nslookup google.com टाइप करके किसी भी डोमेन का IP एड्रेस पता कर सकते हैं।

6. अन्य महत्वपूर्ण प्रोटोकॉल्स

DHCP (Dynamic Host Configuration Protocol) पोर्ट 67/68

DHCP नेटवर्क पर डिवाइसेस को IP एड्रेस असाइन करता है।

उदाहरण:

जब आप अपने लैपटॉप को Wi-Fi से कनेक्ट करते हैं, DHCP निम्नलिखित प्रक्रिया करता है:

  1. DISCOVER: आपका लैपटॉप ब्रॉडकास्ट रिक्वेस्ट भेजता है "क्या कोई DHCP सर्वर है?"
  2. OFFER: राउटर का DHCP सर्वर जवाब देता है "हां, मैं तुम्हें IP 192.168.1.5 दे सकता हूं"
  3. REQUEST: लैपटॉप कहता है "हां, मुझे 192.168.1.5 दीजिए"
  4. ACK: DHCP सर्वर IP एड्रेस और अन्य सेटिंग्स (गेटवे, DNS सर्वर) को कन्फर्म करता है

SSH (Secure Shell) पोर्ट 22

SSH एक सुरक्षित प्रोटोकॉल है जो रिमोट लॉगिन और कमांड एक्जीक्यूशन के लिए उपयोग किया जाता है।

उदाहरण:

वेब डेवलपर्स और सिस्टम एडमिनिस्ट्रेटर्स अक्सर SSH का उपयोग करके रिमोट सर्वर पर लॉगिन करते हैं:

ssh username@server.example.com

इस तरह वे कमांड लाइन से सर्वर की फाइल्स और सेटिंग्स को मैनेज कर सकते हैं।

WebSocket पोर्ट 80/443

WebSocket क्लाइंट और सर्वर के बीच रियल-टाइम, बिड्रिक्शनल कम्युनिकेशन प्रोवाइड करता है।

उदाहरण:

जब आप WhatsApp Web या किसी लाइव चैट एप्लिकेशन का उपयोग करते हैं, तो WebSocket आपके ब्राउज़र और सर्वर के बीच एक ओपन कनेक्शन मेंटेन करता है। इससे बिना पेज रिफ्रेश किए मेसेज रियल-टाइम में अपडेट होते हैं।

जावास्क्रिप्ट में WebSocket कनेक्शन का उदाहरण:

const socket = new WebSocket('ws://example.com/socket');

socket.onopen = () => {
  console.log('Connected!');
  socket.send('Hello Server!');
};

socket.onmessage = (event) => {
  console.log('Message from server:', event.data);
};

निष्कर्ष:

इंटरनेट प्रोटोकॉल्स डिजिटल कम्युनिकेशन की भाषा हैं। ये अलग-अलग उद्देश्यों के लिए अलग-अलग तरीके से काम करते हैं, लेकिन सभी का लक्ष्य एक ही है: विभिन्न डिवाइसेस के बीच डेटा का सुरक्षित और विश्वसनीय आदान-प्रदान।

वेब डेवलपमेंट में करियर बनाने के लिए, इन प्रोटोकॉल्स की बेसिक समझ होना महत्वपूर्ण है, विशेष रूप से HTTP, HTTPS, और WebSocket.
