Domain Names, Hosting, DNS and Browsers

1. Domain Name क्या है?

Domain Name एक इंटरनेट पर वेबसाइट का पता होता है, जिसे याद रखना आसान होता है। यह आईपी एड्रेस (जैसे 192.168.1.1) की जगह इस्तेमाल किया जाता है क्योंकि नंबर्स याद रखना मुश्किल होता है।

Domain Name के उदाहरण:

  • google.com
  • facebook.com
  • wikipedia.org

Domain Name के हिस्से:

एक Domain Name के अलग-अलग हिस्से होते हैं:

  • Top-Level Domain (TLD): डोमेन नाम का आखिरी हिस्सा (.com, .org, .net, .in)
  • Second-Level Domain: TLD से पहले का हिस्सा (google, facebook, wikipedia)
  • Subdomain: मुख्य डोमेन से पहले का हिस्सा (mail.google.com में "mail" एक subdomain है)
ध्यान दें: Domain Name खरीदना पड़ता है और इसके लिए हर साल फीस देनी पड़ती है। इसे कंपनियां जैसे GoDaddy, Namecheap से खरीद सकते हैं।

2. Hosting क्या है?

Hosting एक सेवा है जहां आपकी वेबसाइट के सभी फाइल्स, डेटा और कोड रखे जाते हैं। ये एक कंप्यूटर (सर्वर) पर रहते हैं जो 24/7 इंटरनेट से जुड़ा रहता है, ताकि दुनिया में कोई भी, कभी भी आपकी वेबसाइट देख सके।

Hosting के प्रकार:

  • Shared Hosting: एक सर्वर पर कई वेबसाइट्स होती हैं (सबसे सस्ता)
  • VPS (Virtual Private Server): एक सर्वर को कई छोटे वर्चुअल सर्वर में बांटा जाता है
  • Dedicated Hosting: पूरा सर्वर सिर्फ आपकी वेबसाइट के लिए (सबसे महंगा)
  • Cloud Hosting: कई सर्वर मिलकर आपकी वेबसाइट होस्ट करते हैं

Popular Hosting Companies:

  • Bluehost
  • HostGator
  • SiteGround
  • GoDaddy
  • AWS (Amazon Web Services)
ध्यान दें: डोमेन नेम और होस्टिंग अलग-अलग सेवाएं हैं। आप एक जगह से डोमेन और दूसरी जगह से होस्टिंग खरीद सकते हैं।

3. DNS (Domain Name System) क्या है और कैसे काम करता है?

DNS इंटरनेट का फोन बुक है। इसका काम है डोमेन नेम (जैसे google.com) को IP एड्रेस (जैसे 142.250.192.46) में बदलना। जब आप ब्राउज़र में एक वेबसाइट का नाम टाइप करते हैं, तो DNS उस नाम को सही IP एड्रेस में बदलता है।

DNS कैसे काम करता है:

  1. आप ब्राउज़र में "google.com" टाइप करते हैं
  2. आपका कंप्यूटर सबसे पहले अपनी मेमोरी में देखता है (DNS कैश)
  3. अगर वहां नहीं मिलता, तो आपका ISP के DNS सर्वर को पूछता है
  4. अगर वहां भी नहीं मिलता, तो Root DNS सर्वर से पूछा जाता है
  5. Root DNS सर्वर TLD DNS सर्वर (.com) का पता देता है
  6. TLD DNS सर्वर google.com के Authoritative DNS सर्वर का पता देता है
  7. Authoritative DNS सर्वर google.com का सही IP एड्रेस देता है
  8. अब आपका कंप्यूटर सीधे उस IP एड्रेस से कनेक्ट करता है
ध्यान दें: DNS काम ज्यादातर मामलों में मिलीसेकंड में पूरा हो जाता है, इसलिए आपको यह प्रोसेस दिखाई नहीं देता।

DNS कैश क्या है?

DNS कैश एक तरह का अस्थायी स्टोरेज होता है, जिसमें पहले किए गए DNS लुकअप के रिजल्ट सेव किए जाते हैं। यानी जब आप किसी वेबसाइट पर जाते हैं, तो उस वेबसाइट का डोमेन नेम और IP एड्रेस एक समय के लिए आपके डिवाइस की मेमोरी में सेव हो जाता है।

DNS कैश के प्रकार:

  1. ब्राउज़र DNS कैश: आपके वेब ब्राउज़र (Chrome, Firefox, आदि) में होता है
  2. ऑपरेटिंग सिस्टम DNS कैश: आपके कंप्यूटर के ऑपरेटिंग सिस्टम (Windows, macOS, Linux) में होता है
  3. राउटर DNS कैश: आपके इंटरनेट राउटर में होता है
  4. ISP DNS कैश: आपके इंटरनेट सेवा प्रदाता के सर्वर पर होता है

DNS कैश के फायदे:

  • तेज इंटरनेट स्पीड: हर बार DNS लुकअप नहीं करना पड़ता, जिससे पेज लोडिंग टाइम कम होता है
  • नेटवर्क ट्रैफिक कम: DNS सर्वर्स पर कम लोड पड़ता है
  • इंटरनेट स्थिरता में सुधार: अगर DNS सर्वर डाउन हो जाए तो भी आप कुछ वेबसाइट्स पर जा सकते हैं

DNS कैश के नुकसान:

  • पुरानी जानकारी: अगर किसी वेबसाइट का IP एड्रेस बदल गया है लेकिन कैश में पुराना एड्रेस है, तो कनेक्शन प्रॉब्लम हो सकती है
  • DNS पॉइज़निंग: हैकर्स पुराने कैश का फायदा उठाकर आपको गलत वेबसाइट पर रीडायरेक्ट कर सकते हैं

DNS कैश को कैसे क्लियर करें:

Windows में:

1. CMD (कमांड प्रॉम्प्ट) खोलें
2. टाइप करें: ipconfig /flushdns
3. Enter दबाएं

macOS में:

1. टर्मिनल खोलें
2. टाइप करें: sudo killall -HUP mDNSResponder
3. Enter दबाएं और पासवर्ड डालें

ब्राउज़र कैश क्लियर करना:

Chrome: Settings > Privacy and Security > Clear browsing data > Cached images and files
Firefox: Options > Privacy & Security > Cookies and Site Data > Clear Data

TTL (Time To Live) क्या है?

हर DNS रिकॉर्ड के साथ एक TTL वैल्यू होती है, जो बताती है कि उस रिकॉर्ड को कैश में कितने समय तक रखा जा सकता है। TTL सेकंड्स में होती है, जैसे 300 (5 मिनट), 3600 (1 घंटा), 86400 (1 दिन)। TTL खत्म होने के बाद सिस्टम फ्रेश DNS लुकअप करता है।

उदाहरण: जब आप netflix.com पर जाते हैं पहली बार, आपका कंप्यूटर DNS लुकअप करता है और रिजल्ट कैश में सेव करता है। अगर इस रिकॉर्ड का TTL 1 घंटा है, तो अगले 1 घंटे तक जब भी आप netflix.com पर जाएंगे, आपका कंप्यूटर DNS सर्वर से पूछे बिना कैश से जानकारी लेगा।

4. ब्राउज़र क्या है और कैसे काम करता है?

ब्राउज़र एक सॉफ्टवेयर है जो इंटरनेट पर वेबसाइट्स को दिखाता है। जैसे Chrome, Firefox, Safari, Edge आदि। यह HTML, CSS, और JavaScript को पढ़कर उसे आपके स्क्रीन पर सुंदर तरीके से दिखाता है।

ब्राउज़र कैसे काम करता है:

  1. URL Processing: आप जो URL (जैसे google.com) टाइप करते हैं, उसे ब्राउज़र समझता है
  2. DNS Lookup: ब्राउज़र DNS से उस डोमेन का IP एड्रेस मांगता है
  3. TCP Connection: ब्राउज़र उस IP एड्रेस से कनेक्ट करता है
  4. HTTP Request: ब्राउज़र सर्वर से वेबपेज मांगता है
  5. Server Response: सर्वर वेबपेज भेजता है (HTML, CSS, JS, इमेज आदि)
  6. Rendering: ब्राउज़र मिली फाइल्स को प्रोसेस करता है:
    • HTML: पेज का स्ट्रक्चर बनाता है
    • CSS: पेज को स्टाइल देता है
    • JavaScript: पेज को इंटरैक्टिव बनाता है
  7. Display: पूरा पेज आपके स्क्रीन पर दिखाई देता है

Popular Browsers:

  • Google Chrome
  • Mozilla Firefox
  • Safari
  • Microsoft Edge
  • Opera
ध्यान दें: अलग-अलग ब्राउज़र में वेबपेज अलग-अलग दिख सकते हैं, क्योंकि हर ब्राउज़र HTML, CSS और JavaScript को थोड़ा अलग तरीके से प्रोसेस करता है।

5. DNS सर्वर और IP एड्रेस - विस्तृत जानकारी

DNS सर्वर के प्रकार

  1. Root DNS सर्वर:
    • पूरी दुनिया में सिर्फ 13 रूट सर्वर हैं
    • ये TLD (.com, .org, .net) के DNS सर्वर का पता बताते हैं
    • उदाहरण: a.root-servers.net, b.root-servers.net
  2. TLD DNS सर्वर:
    • हर TLD (.com, .org, .in) का अपना DNS सर्वर होता है
    • ये डोमेन के Authoritative DNS सर्वर का पता बताते हैं
    • उदाहरण: a.gtld-servers.net (.com के लिए)
  3. Authoritative DNS सर्वर:
    • ये वास्तविक डोमेन का IP एड्रेस रखते हैं
    • हर वेबसाइट का मालिक अपना Authoritative DNS सर्वर चुनता है
    • उदाहरण: ns1.google.com, ns2.google.com

DNS कमांड्स और उपयोग

1. nslookup कमांड

यह कमांड DNS लुकअप के लिए उपयोग किया जाता है:

# डोमेन का IP एड्रेस पता करें
nslookup google.com

# विशेष DNS सर्वर से पूछें
nslookup google.com 8.8.8.8

# MX रिकॉर्ड देखें (मेल सर्वर)
nslookup -type=mx google.com

# NS रिकॉर्ड देखें (नेम सर्वर)
nslookup -type=ns google.com

2. dig कमांड (Linux/Mac)

यह कमांड DNS जानकारी के लिए अधिक विस्तृत जानकारी देता है:

# सामान्य DNS लुकअप
dig google.com

# विशेष DNS सर्वर से पूछें
dig @8.8.8.8 google.com

# सभी DNS रिकॉर्ड देखें
dig google.com ANY

# TTL वैल्यू देखें
dig google.com +ttl

3. ping कमांड

यह कमांड नेटवर्क कनेक्टिविटी चेक करने के लिए उपयोग किया जाता है:

# IP एड्रेस पिंग करें
ping 8.8.8.8

# डोमेन पिंग करें
ping google.com

# विशेष पैकेट साइज के साथ
ping -l 1000 google.com

DNS रिकॉर्ड्स के प्रकार

रिकॉर्ड टाइप उपयोग उदाहरण
A IPv4 एड्रेस google.com → 142.250.192.46
AAAA IPv6 एड्रेस google.com → 2404:6800:4002:80c::200e
MX मेल सर्वर google.com → aspmx.l.google.com
NS नेम सर्वर google.com → ns1.google.com
CNAME अलियास www.google.com → google.com

DNS कैश क्लीयर करने के कमांड्स

Windows में:

# DNS कैश क्लीयर करें
ipconfig /flushdns

# DNS कैश देखें
ipconfig /displaydns

Linux में:

# systemd-resolved का उपयोग करने वाले सिस्टम में
sudo systemd-resolve --flush-caches

# dnsmasq का उपयोग करने वाले सिस्टम में
sudo service dnsmasq restart

Mac में:

# DNS कैश क्लीयर करें
sudo killall -HUP mDNSResponder
महत्वपूर्ण बिंदु:
  • DNS सर्वर हायरार्की में काम करते हैं
  • हर DNS रिकॉर्ड का एक TTL (Time To Live) होता है
  • DNS कैश स्पीड बढ़ाता है लेकिन कभी-कभी प्रॉब्लम भी कर सकता है
  • अलग-अलग DNS सर्वर अलग-अलग स्पीड दे सकते हैं
  • Google DNS (8.8.8.8) और Cloudflare DNS (1.1.1.1) सबसे पॉपुलर हैं

सारांश: इंटरनेट कैसे काम करता है

  1. आप ब्राउज़र में एक डोमेन नेम (www.example.com) टाइप करते हैं
  2. DNS उस डोमेन नेम को IP एड्रेस में बदलता है
  3. ब्राउज़र उस IP एड्रेस पर होस्टिंग सर्वर से कनेक्ट करता है
  4. होस्टिंग सर्वर वेबसाइट का कंटेंट भेजता है
  5. ब्राउज़र उस कंटेंट को आपके स्क्रीन पर दिखाता है
